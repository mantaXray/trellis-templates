# Debug Symptom Index Pattern

> 本文件不是具体项目的 debug 手册，而是给项目本地知识库使用的症状索引模板。

## 建议格式

```markdown
## <症状>

- 先看：<相关日志 / 文件 / 外设 / 任务>
- 常见根因：
  - <根因 1>
  - <根因 2>
- 对应 pitfall：`PIT-XXX-000`
- 验证命令：
  - <命令或手工验证步骤>
- 不要做：
  - <容易误判或破坏现场的操作>
```

## 使用规则

- 症状索引适合放项目本地 `doc/` 或项目 `.trellis/spec/`。
- 模板仓库只保留通用格式和通用 pitfall，不保存项目现场复盘。
- 如果同一症状在多个 MCU 项目重复出现，把共性规则反哺到 `firmware/pitfalls.md`。
